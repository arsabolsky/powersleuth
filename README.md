# PowerSleuth

A lightweight macOS menubar app that monitors battery drain over time and tells you **exactly why** your battery life is what it is.

![macOS 13+](https://img.shields.io/badge/macOS-13%2B-blue)
![Swift](https://img.shields.io/badge/Swift-5.9-orange)
![License](https://img.shields.io/badge/license-MIT-green)

## Why PowerSleuth?

Other battery apps (coconutBattery, Stats, AlDente) show you the *current state* of your battery. PowerSleuth **studies your Mac over time** and diagnoses *why* you're getting the battery life you're getting:

- **Sleep drain** — Is your Mac staying partially awake when the lid is closed? Which process is keeping it alive?
- **Active drain rate** — How many Watts are you pulling right now, and is that normal?
- **Thermal impact** — Is your Mac running hot, making the battery drain faster?
- **Battery health** — Is aging capacity the root cause of your shorter range?

## Features

- **Menubar icon** showing real-time power draw in Watts
- **Popover** with current stats and active sleep-prevention processes
- **History charts** — drain rate and battery level over 24h or 7d (using Apple Charts)
- **Analysis engine** — rule-based diagnosis in plain English
- **Battery health tracking** — capacity retention % and cycle count
- **Lightweight** — 30s polling interval, SQLite storage, ~30MB RAM
- **No root required** — uses only public macOS APIs

## Data Sources

| Metric | Source | Root? |
|--------|--------|-------|
| Battery %, voltage, current | IOPowerSources | No |
| Temperature, cycle count | IORegistry/AppleSmartBattery | No |
| Power draw (Watts) | Computed: `|mA × mV| / 1M` | No |
| Sleep prevention | `pmset -g assertions` | No |
| Thermal state | `NSProcessInfo.thermalState` | No |
| Sleep/wake events | `NSWorkspace` notifications | No |

## Requirements

- macOS 13.0 (Ventura) or later
- Xcode 15+

## Installation

### Build from source

```bash
git clone https://github.com/arsabolsky/powersleuth.git
cd powersleuth
make setup     # installs xcodegen (via Homebrew)
make install   # builds Release and copies to /Applications
```

Then launch **PowerSleuth** from Spotlight or `/Applications`.

### Open in Xcode

```bash
make open
```

## Architecture

```
PowerSleuth/
├── Models/          # BatterySnapshot, PowerAssertion, DrainSession, DrainDiagnosis
├── Services/        # BatteryMonitor, AssertionMonitor, SessionTracker,
│                    #   DatabaseService (GRDB/SQLite), AnalysisEngine
└── Views/           # MenuBarView, StatusLabel, DashboardView,
                     #   HistoryChartView, AnalysisView
```

Data is stored in `~/Library/Application Support/PowerSleuth/powersleuth.db` with a 30-day rolling window.

## Analysis Engine

The engine diagnoses drain by checking (in priority order):

1. **Battery health < 80%** — capacity degradation is a primary cause
2. **Sleep drain > 2%/hr** — identifies which process prevented deep sleep
3. **Thermal stress** — Serious/Critical state correlates with high draw
4. **Low Power Mode** — flags when it's active and helping
5. **High active workload** — elevated watts with no other known cause

## Privacy

PowerSleuth runs entirely on-device. No data leaves your Mac. No analytics, no telemetry.
